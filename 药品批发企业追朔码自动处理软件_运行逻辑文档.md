## 一、核心算法：统一排重模式（v2.4）

### 1.1 为何统一

v2.3 分"全1级码模式"和"混装模式"两套路径，需要用户判断数据特征且父码子码同时在输入时产生重复输出。v2.4 全部走统一流程，自动判断。

### 1.2 核心数据结构

| 名称 | 类型 | 作用 |
|:-----|:-----|:------|
| `输入表追溯码集合` | `set(str)` | 原始输入文件中的所有追溯码，**用于1级码的同箱匹配** |
| `covered_set` | `set(str)` | 已处理过的码，跳过重复 |
| `output_1level_set` | `set(str)` | 已输出的1级码，防止重复输出 |

### 1.3 双路径规则（核心）

```
输入记录按码包装数降序排列（非1级码 → 1级码）

FOR 每条输入记录:
    IF 此码 ∈ covered_set: 跳过（已被覆盖）
    
    IF 码包装数 ≠ "1" 且 ≠ "?":
        ── 非1级码路径 ──
        query_single(码) → 一键复制所有码
        提取全部下级1级码
        过滤已输出的 → 输出新的
        covered_set += 全部下级码（整箱标记）
    
    IF 码包装数 == "1":
        ── 1级码路径 ──
        query_single(码) → 一键复制所有码
        提取同箱全部1级码
        与【输入表追溯码集合】对比 → 只保留重复项
        过滤已输出的 → 输出新的
        covered_set += 仅实际输出的码（不覆盖整箱）
```

### 1.4 两条路径的关键区别

| | 非1级码 | 1级码 |
|:--|:--------|:------|
| covered_set 更新 | **整箱全部下级码** → 同箱后续行全部跳过 | **只加实际输出的码** → 同箱不同批号不会误跳 |
| 输出 | 全部下级1级码（过滤已输出） | 仅保留「也是在输入表中」的1级码 |

## 二、推演：真实极端场景

### 2.1 输入数据

| 药品 | 批号 | 输入内容 | 行数 | 说明 |
|:----|:----:|:---------|:----:|:-----|
| 麝香保心丸 | A | 3级码 × 2 | 2 | 每箱450盒（1个3级码→多个2级码组→共450个1级码） |
| 麝香保心丸 | B | 2级码×13 + 1级码×5 | 18 | 共200盒：13箱各15盒+5个散货 |
| 阿莫西林 | A | 1级码×100 | 100 | 散货，原箱容量160盒 |
| 阿莫西林 | B | 1级码×80 | 80 | 散货，原箱容量160盒，与A混装在同一箱 |
| **合计** | | **200行** | | **期望输出：1280个1级码** |

输入表集合 = `{3A1, 3A2, 2B1~2B13, 1B1~1B5, 1AA1~1AA100, 1AB1~1AB80}`

### 2.2 逐轮推演

#### 第1轮：3A1（3级码，麝香A，非1级）

| 步骤 | 结果 |
|:----|:------|
| query_single(3A1) | 返回 {3A1, 下级2级码, Z001~Z450} |
| 非1级路径 → 输出全部 | **Z001~Z450（450个）**，批号A ✅ |
| covered_set += **整箱** | +3A1, +下级2级码, +Z001~Z450 |

#### 第2轮：3A2（3级码，麝香A，非1级）

| 步骤 | 结果 |
|:----|:------|
| query_single(3A2) | 返回 {3A2, 下级2级码, Z451~Z900} |
| 非1级路径 → 输出全部 | **Z451~Z900（450个）**，批号A ✅ |
| covered_set += **整箱** | +3A2, +下级2级码, +Z451~Z900 |

> 批号A累计输出 900个 ✅

#### 第3~15轮：2B1~2B13（2级码，麝香B，非1级）

逐个 query_single，每个输出下级15个1级码 → **195个** ✅，批号B

covered_set += 各箱整箱码

#### 第16轮：1B1（1级码，麝香B，散货）

| 步骤 | 结果 |
|:----|:------|
| 1B1在covered_set? | 不在 → 需查询 |
| query_single(1B1) | 返回同箱 {2级码B箱, Z901~Z915} |
| **1级路径 → 与输入表对比** | Z901在输入表 → 保留；Z902~Z915不在 → **排除** |
| 输出 | **1个**（1B1自身）✅ |
| covered_set += **仅Z901** | 不覆盖整箱 |

#### 第17~20轮：1B2~1B5

如果已被2B级的整箱覆盖 → 跳过；如果未被覆盖 → 各自重复杂比流程，输出自身

> 批号B累计 200个 ✅

#### 第21轮：1AA1（1级码，阿莫西林A）

| 步骤 | 结果 |
|:----|:------|
| query_single(1AA1) | 返回 {阿莫西林A箱, ZAA1~ZAA160} |
| **1级路径 → 与输入表对比** | ZAA1~ZAA100在输入表 → **保留100个**；ZAA101~ZAA160不在 → 排除 |
| 输出 | **100个** ✅，批号A |
| covered_set += **仅ZAA1~ZAA100** | 不覆盖ZAA101~ZAA160 |

#### 第22~120轮：1AA2~1AA100

在covered_set → 跳过 ✅

#### 第121轮：1AB1（1级码，阿莫西林B）

| 步骤 | 结果 |
|:----|:------|
| 1AB1在covered_set? | **不在！**（上轮只加了ZAA1~ZAA100，没加ZAB区间）✅ |
| query_single(1AB1) | 返回同箱 {2级码, ZAB1~ZAB160} |
| 与输入表对比 | ZAB1~ZAB80保留；其余排除 |
| 输出 | **80个** ✅，批号B |
| covered_set += 仅ZAB1~ZAB80 |

#### 第122~200轮：1AB2~1AB80

在covered_set → 跳过 ✅

### 2.3 结果

| 来源 | 个数 | 批号 |
|:----|:----:|:----:|
| 3A1 / 3A2 拆箱 | 900 | 麝香A ✅ |
| 2B1~2B13 拆箱 | 195 | 麝香B ✅ |
| 1B1~1B5 输入匹配 | 5 | 麝香B ✅ |
| 1AA1 输入匹配 | 100 | 阿莫西林A ✅ |
| 1AB1 输入匹配 | 80 | 阿莫西林B ✅ |
| **总计** | **1280** | **无重复无遗漏** ✅ |

## 五、异常处理

| 场景 | 处理 |
|:----|:-----|
| 查询为空 | 重试3次 |
| 批号为空 | 内部重试2次 |
| 3次全失败 | 弹窗：跳过/重试/停止 |
| 空白包装数 | 重读1次→仍空白→标记"?"→按非1级码处理 |
| 焦点丢失 | 每步操作前激活窗口 |
| F1+F1 | 全局紧急停止 |
| FAILSAFE | 鼠标移至角落急停 |
| 崩溃续跑 | _progress.json 断点续跑 |
