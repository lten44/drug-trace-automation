# 国瑞新特药追朔码自动处理软件

## 软件介绍

**软件名称**：国瑞新特药追朔码自动处理软件  
**当前版本**：v2.2  
**开发语言**：Python 3.12  
**运行平台**：Windows 10/11  
**打包工具**：PyInstaller（单文件 exe）

本软件专为药品批发企业设计，用于自动化处理药品追溯码。通过模拟鼠标键盘操作与"码上放心"客户端交互，实现追溯码的批量查询、1级码自动提取和标准化表格输出，大幅提升日常工作效率。

---

## 一、开发背景与需求分析

### 1.1 业务痛点

在日常药品流通管理中，批发企业需要将Excel表格中的追溯码（3级码）转换为1级码，以便向医院提供标准化的追溯码清单。传统手工操作流程如下：

1. 打开Excel表格，逐个复制追溯码
2. 切换到"码上放心"客户端
3. 在"关联关系查询"页面粘贴追溯码并查询
4. 等待查询结果显示后，点击"一键复制所有码"
5. 将复制结果粘贴到Word文档中
6. 使用通配符筛选出1级码
7. 整理成标准格式输出到Excel

**问题**：每天处理量可能达到数千条追溯码，手工操作耗时极长且容易出错。

### 1.2 技术方案选型

| 方案 | 可行性 | 结论 |
|------|--------|------|
| 码上放心开放API | ❌ 批发企业无法获取零售药店接口权限 | 不可行 |
| WES接口 | ⚠️ 需年费2800元 | 成本过高 |
| pywinauto控件定位 | ❌ 码上放心基于WebView（Chromium），无法识别控件 | 不可行 |
| **pyautogui坐标模拟** | ✅ 通过屏幕坐标校准实现自动化 | **最终方案** |

最终采用 **pyautogui 屏幕坐标模拟** 方案，通过校准系统记录关键按钮位置，实现鼠标点击自动化。

## 二、核心设计思路

### 2.1 整体架构

```
┌─────────────┐    ┌──────────────┐    ┌─────────────┐    ┌──────────────┐
│  Excel输入   │───▶│  读取解析     │───▶│  自动查询    │───▶│  生成输出    │
│  (.xlsx)     │    │  (openpyxl)  │    │  (pyautogui) │    │  (.xlsx)     │
└─────────────┘    └──────────────┘    └─────────────┘    └──────────────┘
                                              │
                                              ▼
                                       ┌──────────────┐
                                       │  码上放心客户端  │
                                       │  (WebView)    │
                                       └──────────────┘
```

### 2.2 屏幕坐标校准系统

由于"码上放心"是WebView应用，传统控件定位方案失效。因此设计了**坐标校准系统**：

- 用户通过校准窗口，依次点击需要操作的界面元素
- 程序记录每个元素的屏幕坐标（x, y）
- 后台线程实时显示鼠标位置，按 F5 热键记录坐标
- 校准数据保存到 `calibration.json`，下次启动自动加载

**校准点（共5个）**：

| 序号 | 校准点 | 用途 |
|------|--------|------|
| 1 | 追溯码输入框 | 粘贴追溯码 |
| 2 | 查询按钮 | 触发查询 |
| 3 | 一键复制所有码（首次位置） | 查询前复制按钮位置 |
| 4 | 一键复制所有码（查询后位置） | 查询后按钮位置（会变化） |
| 5 | 产品批号文字位置 | 双击选中并复制批号 |

### 2.3 1级码过滤算法

从"码上放心"复制出来的内容格式如下：

```
3级码
87170610000179143429
2级码
87170620006991633042
87170620006991633043
...（多个2级码）
1级码
83144190035120555493
83144190035112446711
...（多个1级码）
```

过滤逻辑：
1. 找到"1级码"标记
2. 持续收集后面所有纯数字行（16-24位数字）
3. 遇到其他级别标记（2级码、3级码等）时停止收集

### 2.4 智能跳过机制（v2.1新增）

关键发现：**原表格中"码包装数"列为1的行，本身就是1级码**，无需再查询关联关系。

处理逻辑优化：
- 码包装数 = 1 → 跳过查询和过滤步骤，只查询获取产品批号
- 码包装数 ≠ 1 → 完整执行查询→复制→过滤→获取批号流程

**效果**：如果表格中大部分行已是1级码，处理时间可缩短50%以上。

## 三、功能模块详解

### 3.1 Excel读取模块

- 按表头名称查找列索引（兼容不同列顺序的输入表格）
- 自动解析前4行元信息（扫码日期、单据类型、单据号、收货单位名称）
- 第6行为表头，第7行起为数据行
- 每行提取13个字段：追溯码、药品名称、剂型规格、包装规格、批准文号、生产厂家、码类型、码包装数、零头数、生产信息、产品批号、验证信息、扫描时间

### 3.2 自动查询模块

每个追溯码的处理流程：

```
输入追溯码 → 点击查询 → 等待结果 → 点击复制 → 读取剪贴板 → 过滤1级码 → 获取批号
```

- 使用后台线程处理，不阻塞UI
- 支持随时停止（通过线程事件标志）
- 实时进度反馈（进度条 + 状态文字）

### 3.3 Excel输出模块

输出表格结构：

| 行 | 内容 |
|----|------|
| 1-4 | 元信息（扫码日期、单据类型、单据号、收货单位名称） |
| 6 | 表头 |
| 7+ | 数据行 |

输出列（10列）：

| 列号 | 列名 | 说明 |
|------|------|------|
| 1 | 追溯码 | 查询到的1级码（文本格式） |
| 2 | 药品器械名称 | 从输入表格继承 |
| 3 | 剂型规格 | 从输入表格继承 |
| 4 | 包装规格 | 从输入表格继承 |
| 5 | 批准文号 | 从输入表格继承 |
| 6 | 生产厂家 | 从输入表格继承 |
| 7 | 码类型 | 从输入表格继承 |
| 8 | 码包装数 | 固定为"1" |
| 9 | 产品批号 | 从码上放心查询获取 |
| 10 | 扫描时间 | 从输入表格继承 |

样式：等线字体、左对齐、无边框、列宽自适应。

### 3.4 自动更新模块

- 程序启动时自动检查同目录下的 `version.json`
- 比较版本号，如发现新版本则自动更新
- 更新流程：重命名旧exe为 `.old` → 复制新exe → 启动新版本 → 退出旧版本
- 解决了Windows下运行中的exe无法直接覆盖的问题

### 3.5 文件命名规则

```
汕头国瑞新特药-{收货单位}-{药品名1}&{药品名2}&...-{单据号}.xlsx
```

示例：`汕头国瑞新特药-汕头市澄海区中医医院-丹七胶囊&六味地黄丸-XC2605280001.xlsx`

**修复**：改为找到"1级码"标记后，持续收集后续所有纯数字行，直到遇到其他级别标记。

尝试复制原始Excel保留样式，但行数变化时处理复杂，后续版本回退为新建表格。

**新增**：

## 七、使用说明

### 7.1 首次使用

1. **校准位置**：点击"校准位置"按钮，按提示依次点击5个位置（按 F5 记录坐标）
2. **选择文件**：点击"浏览"选择要处理的Excel文件
3. **开始处理**：点击"开始处理"，程序自动运行
4. **查看结果**：处理完成后在导出目录查看生成的Excel文件

### 7.2 日常使用

校准数据会自动保存，日常使用只需选择文件并点击"开始处理"即可。

### 7.3 更新升级

将新版 exe 和更新后的 version.json 放入软件目录，下次打开程序时自动更新。

## 八、已知限制与未来展望

### 已知限制

1. **屏幕分辨率依赖**：校准坐标基于固定分辨率，更换显示器或调整缩放比例后需要重新校准
2. **单线程查询**：逐个查询追溯码，无法并行处理
3. **界面变化敏感**：码上放心客户端界面更新后可能需要重新校准

### 未来展望

1. **OCR识别**：使用图像识别自动定位按钮，减少手动校准
2. **多线程查询**：同时查询多个追溯码，进一步提升效率
3. **批量导入**：支持拖拽文件夹自动处理多个Excel文件
4. **数据校验**：增加追溯码格式校验，提前发现异常数据
