## 一、核心架构重构：统一排重模式

### 🔴 改动前（v2.3）

两套独立的处理路径：

```
全1级模式（码包装数全为1）  ──→ 只查第1行批号，复用给全部
混装模式（存在非1级码）    ──→ 逐行顺序完整查询
```

**问题**：父码（3级）和子码（2级/1级）同时在输入表中时，查两次 → 输出重复1级码

### 🟢 改动后（v2.4）

一条统一路径：

```
全部码 → 按包装数降序排列 → 逐个处理
                            │
                     covered_set 排重
                     output_1level_set 防重复输出
```

**效果**：父码查询结果中的全量码（3+2+1级）全部加入 covered_set → 子码直接跳过 → 零重复

## 三、新增未实装优化（5项）

### 1. 5秒倒计时 ⏳

**改动前**：确认弹窗后直接开始操作

**改动后**：确认弹窗 → 状态栏显示 5-4-3-2-1 倒计时 → 开始操作

**代码**：`_start_process()` 中 `for i in range(5, 0, -1):`

### 2. 查询失败弹窗报警 🚨

**改动前**：3次重试失败后静默返回 error，用户无感知

**改动后**：弹出居中对话框，三个选项：

| 按钮 | 作用 |
|------|------|
| 跳过 | 跳过当前码，继续处理下一个 |
| 重试 | 再重试一次 |
| 停止 | 中断整个处理流程 |

**代码**：新增 `_show_fail_dialog()` → `CodeQueryUI` 通过 `fail_callback` 参数调用

### 3. 每步重新激活窗口 🪟

**改动前**：仅在开始处理时激活一次码上放心窗口

**改动后**：`CodeQueryUI._click()` 和 `_type_code()` 每次操作前都先调用 `_activate_window()`

**代码**：新增 `CodeQueryUI._activate_window()` 方法

### 4. 空白码包装数处理 📄

**改动前**：码包装数为空时不处理，后续可能报错

**改动后**：空值时重读1次单元格 → 仍空白 → 标记为 `?`（异常码）→ 走完整查询流程

**代码**：`read_sales_excel()` 中新增空白检测 + 重读逻辑

### 5. F5全局校准双重保险 🔑

**改动前**：仅使用 pynput 库监听 F5，部分环境不可靠

**改动后**：pynput + Windows API `GetAsyncKeyState` 双重监听

**代码**：新增 `CalibrateWindow._api_f5_monitor()` WinAPI 轮询线程
